Orion! Library : 3DEngine Flags description (Version 4.x)
Last Update : 01/06/2002

 

General Guidelines:

- Objects can have no textures. They are rendered as white, so they can take a full color with associated VertexColor.
- Do not use more than two or three simultaneous Dynamic light.
- Prefer using models > 200 Polygons. (>1000 polygons is good, MAXIMUM IS 65535 Vertices / Meshe)
- Prefer having between 10.000 and 15.000 simultaneous visible polygons. More is a frame rate killer

 

Max4 Supported Features :

- Multi-Sub materials are supported. Use with care, try to avoid small subset of faces.
- Hierarchy between meshes, lights and helpers
- Color keying for lights
- Texture Alphamapping/Environement Mapping
- Bones / Skinning
- Material parameters are used in case of dynamic ligthing

Flag Description :

Mesh Modifiers :
.NoCull : Disables the backface culling. Useful for single-faced meshes.
.NoDynaLight : Disables the effect of dynamic lighting.
.Hide : Permanently hides a mesh from the scene. Useful for Meshe Modifier based layers.
.NoFog : Disables the effect for scene fog.
.Floor : Allows the floor to "Reflect". In fact, you must duplicate the objects to be mirrored by
rotating them by 180°.
.Alpha : Meshe is alphablended
.AlphaMap : Meshe contains an alpha channel in Texture.
.Sprite : Meshe will always face the camera. Useful for planes intended to be sprites.



Note on AlphaMap flag :
The Specular component of the associated material is used as alpha map for the objects with this flag.
If there is no specular map when specifying this flag, the Alpha Channel is ignored.
The alpha map file name MUST contain the string "alpha" to be treated as alpha channel. (ie. : AlphaChan0.jpg)

.Mirror : Meshe is a mirror (Should be a plane)
.EnvMap : Meshe has envmap, must have two textures the second is the reflected one.
.NoLight : Overrides the vertex color to full white.
.MorphSrc.fX : First object for morphing. X is the start frame.
.MorphDest.fX.cY : Next object for morphing. X is the start frame. Y is the object count.

Notes on morping :
The two objets MUST have the same number of vertices and the same prefixed name.

Example :
Object 1 : "Sphere01.MorphSrc.f0"
Object 2 : "Sphere01.MorphDest.f100.c0"
Object 3 : "Sphere01.MorphDest.f200.c1"

Object morphes from 'Object 1' to 'Object 2' through frame 0 to 100, then from 'Object 2' to
'Object 3' through frame 100 to 200.

.RootLod.dX : Root mesh for LOD. X is the mininal activation distance
.Lod.dX : Child meshe for Lod. X is the mininal activation distance

Notes on LOD :
The different child meshes must have the same prefixed name.
The parameter X is the where the level is activated (Mesh center<->Camera distance)

Example :
Object 1 : "Sphere01.RootLod.d100" (Max details)
Object 2 : "Sphere01.Lod.d200"
Object 3 : "Sphere01.Lod.d300"
Object 4 : "Sphere01.Lod.d400" (Min details)

In this example, the object #1 is the most detailed. Is it active when the distance between the center
of the mesh and the camera position is below or equal to 100. Object #2 is active when distance is between
100 and 200 and so on. If distance is over 400, the object #4 is active. Remember that the root object does not
need to be the most detailed.

 

Light Modifiers :

.Halo.m"FileName" : Is a halo, Can have animation track. Must be an OmniLight and must have a texture map file.
.DynaLight : Enables dynamic lighting for current light, Can have animation track, Must be an OmniLight

- .Hide : Permanently hides an object. Useful for mesh-modifier based layers.
- .NoFog : Disables the effect of the fog on meshes.
- .Fog : Enables the fog for a whole scene
